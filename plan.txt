DOMAIN DETECTION RULES:

On every page load:
1. Read window.location.hostname
2. Normalize:
   www.ajio.com -> ajio.com
   m.ajio.com -> ajio.com
   checkout.flipkart.com -> flipkart.com
3. Compare against supported stores list
4. If supported => activate coupon widget
5. If unsupported => stay silent

SHOPPING SITE DETECTION FALLBACK:
If unknown domain, scan page for:
- Add to cart buttons
- Buy now buttons
- Price symbols ₹ $ €
- Cart / checkout text
If strong signals exist:
Show subtle “Looking for deals on this store?” prompt.

COUPON FETCH FLOW:

1. Detect domain
2. Check local cache first
3. If cache missing/stale:
   Fetch remote coupons by domain
4. Save cache locally
5. Render top coupons

API EXAMPLE:

GET /coupons?domain=ajio.com

Response:
{
 "store":"Ajio",
 "coupons":[
   {"code":"SAVE500","title":"₹500 Off","score":82},
   {"code":"NEW20","title":"20% Off","score":75}
 ]
}

COUPON RANKING RULES (NO AI):

Sort by:
1. Highest success_score
2. Not expired
3. Newest created
4. Biggest discount
5. Admin verified source

Formula:
success_score = (worked_count * 2) - failed_count + freshness_bonus

AUTO HIDE RULES:
- expired coupons
- failed ratio too high
- duplicates
- blank codes
- suspicious spam
