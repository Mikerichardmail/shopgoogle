INDIA COUPON EXTENSION - REST API SPECIFICATION
(For implementation with native Supabase endpoints or Custom API)

1. GET /coupons?domain={domain}
Description: Fetch all active top-ranked coupons for a specific shopping domain.
Request: GET
Query Parameters:
- domain (string, required): e.g., 'ajio.com'

Response (Mockup):
{
  "store": "Ajio",
  "coupons": [
    {"id": "uuid1", "code": "SAVE500", "title": "₹500 Off", "score": 82},
    {"id": "uuid2", "code": "NEW20", "title": "20% Off", "score": 75}
  ]
}

2. POST /votes
Description: Submit a user vote indicating if a coupon worked or failed.
Request: POST
Content-Type: application/json
Body:
{
  "coupon_id": "UUID-String",
  "vote": "worked" // or "failed"
}

Response:
{
  "success": true,
  "message": "Vote recorded"
}

3. POST /submissions
Description: Add a user-submitted coupon to the pending tracking queue for review.
Request: POST
Content-Type: application/json
Body:
{
  "store_domain": "ajio.com",
  "code": "FREEBIE",
  "title": "Free socks on rs. 999"
}

Response:
{
  "success": true,
  "submission_id": "UUID-String"
}
